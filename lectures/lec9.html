Alternating List#

Big Goal: Write a function that creates a length \(n\) list of alternating \(3\)’s and \(7\)’s \([3,7,3,7 \dots]\).

Constant List#

Mini Goal: Write a function that takes as input a natural number \(n\) and as output returns a length \(n\) list \([3,3, \dots, 3]\).

Below is the function that we wrote last lecture. We started today by talking a little bit about how the variable n defined in constant(n) is what’s called a local variable. Outside of the function, n is not defined. We saw some examples of this by printing n at various places throughout the cell.

def constant(n):
    mylist = []
    for i in range(n):
        mylist.append(3)
    #print(n)
    return mylist
#print(n)

        mylist[i] = 7

It’s typically not a good idea to copy-paste blocks of code. Notice that we call constant(n) instead of pasting the code for it.

  • If I need to make a change to a part of the code, it’s best to only have to change it in one place.

  • This is the main reason why we encourage you not to copy-paste. Think about it: if there’s a mistake somewhere and you copy-paste it, you’ve now introduced even more mistakes.

alt1(4)

[3, 7, 3, 7]

alt1(5)

[3, 7, 3, 7, 3]

Question from the chat: how do we know when to use () versus [].
Answer: this is one of those things that you’ll just have to get used to with time, but there are a few general rules.

  • In Python, [] are typically used for indexing, and making lists.

  • () are typically used with functions, or to make tuples.

Checking if the index is even or odd#

#this example shows how to use elif statements
#the most elegant way to do this is just with if-else
def alt2(n):
    mylist = []
    for i in range(n):
        #if i is even, append 3
        #else, append 7
        if i%2 == 0:
            mylist.append(3)
        elif i%2 == 1:
            mylist.append(7)
        elif i%2 == 17: #never will happen
            disp("Something is wrong!")
    return mylist

Timing Strats#

The point of this section is to show you that NumPy is very fast.

def alt3(n):
    A = np.zeros(n,dtype=np.int64)+3
    A[1::2] = 7
    return A

Let’s try to write a function that mimics alt3, but doesn’t use NumPy.

def alt4(n):
    mylist = []
    for i in range(n):
        mylist.append(3)
    for i in range(1,n,2):
        mylist[i] = 7
    return mylist

alt4(10)

[3, 7, 3, 7, 3, 7, 3, 7, 3, 7]

alt4(11)

[3, 7, 3, 7, 3, 7, 3, 7, 3, 7, 3]

We’ve already seen examples with Jupyter magics and timing:

%%timeit
alt4(10)

1.31 µs ± 12.3 ns per loop (mean ± std. dev. of 7 runs, 1,000,000 loops each)

What if I only really care about the first number?

import time

This returns the amount of time that’s passed since the beginning of “computer time” (Unix time): January 1st, 1970.

time.time()

1720808306.684022

Notice that if I call it again, some time has passed.

time.time() 

1720808306.691026

Using this concept, we can time how long it takes to run alt4(10**3).

start = time.time()
alt4(10**3)
end = time.time()
t = end - start
print(t)

0.00016117095947265625

I want to find the first \(n\) such that alt4(n) takes more than 5 seconds to run. To do this, we’ll introduce while loops in Python.

n = 100
t = 0
while t < 5:
    n = n*2
    start = time.time()
    alt4(n)
    end = time.time()
    t = end - start
print(n)

52428800

Question: Think about why we call n = n*2 at the start of the loop, and not at the end.

Notice that if I reduce n by half, it takes less than 5 seconds to run.

start = time.time()
alt4(int(n/2))
end = time.time()
t = end-start

t

4.075234651565552

Let’s not compare this to the time it takes for alt3(n).

n

52428800

start = time.time()
alt3(n)
end = time.time()
t = end-start
print(t)

0.3035151958465576

You should stop and appreciate just how fast NumPy is!

Here, we use the NumPy version, but convert the output to a list.

def alt3_list(n):
    A = np.zeros(n,dtype=np.int64)+3
    A[1::2] = 7
    return list(A)

start = time.time()
alt3_list(n)
end = time.time()
t = end-start
print(t)

4.701442241668701

Notice that converting to a list has slowed us down significantly.

Next, we instead use a for-loop to change the odd indices. We’ll see that this also slows us down quite a bit.

def alt3_for(n):
    A = np.zeros(n,dtype=np.int64)+3
    for i in range(1,n,2):
        A[i] = 7
    return A

start = time.time()
alt3_for(n)
end = time.time()
t = end-start
print(t)

4.153294086456299

Takeaways: for-loops and lists are very slow, while NumPy is fast :)
