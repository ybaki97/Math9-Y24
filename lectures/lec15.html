Brief Introduction to Object-Oriented Programming (OOP)#

Very basic idea: we instantiate “objects” and call “methods” on them.

Examples#

import matplotlib.pyplot as plt

Non-OOP way

plt.plot([3,1,4,1])

[<matplotlib.lines.Line2D at 0x7fde8b0edf40>]

OOP way

fig, ax = plt.subplots()
ax.plot([3,1,4,1])

[<matplotlib.lines.Line2D at 0x7fde8b3f52b0>]

Maybe we think that the Non-OOP way is better, because it looks more simple. I’d say for the most simple plots, that’s fine. However, as soon as you want to do anything more complicated, the OOP way is going to be best. We’ll see an example of this later in the lecture.

      ...     plot(x, y[:, col])

      groups::

      datasets. Also this syntax cannot be combined with the *data*

Defining the class is a separate file#

from matplotlib.colors import to_rgb
import numpy as np

class NamedColor():
    def __init__(self, name, rgb_tuple = None):
        if rgb_tuple is None:
            rgb_tuple = [round(255*x) for x in to_rgb(name)]
        self.name = name
        self.rgb = rgb_tuple  
        
    def __str__(self):
        return f"The color {self.name} with RGB value {tuple(self.rgb)}"
    
    def __array__(self,dtype=None):
        return np.array(self.rgb,dtype=dtype)
