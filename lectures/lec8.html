Getting Started with Jupyter Notebook#

This is an example of a markdown cell. This is the kind of cell where I’ll write explanation, upload images, put links, etc. To make a markdown cell, I can do the following:

  • esc

  • b to make a cell below, or a to make a cell above

  • m to change the cell type to a markdown cell.

I highly recommend spending some time in the beginning learning keyboard shortcuts to save you lots of time in the future.

To actually run a cell, you can press cmd+return or shift+return. To enter a cell to edit it, you can press enter.

If you are familiar with HTML, many of the same commands will work in Jupyter Notebook markdown cells. For example:

  • __this__ will produce this

  • _this_ will produce this

  • <font color = red> this </font> will produce this

If you are familiar with \(\LaTeX\), then we can get math to render by wrapping it in $$.

  • $\int_0^1 f(x)dx$ gives \(\int_0^1 f(x)dx\)

  • ${n \choose 2}$ gives \({n \choose 2}\)

Point: Sets are fast to search through.

10000000 #tedious to type :P

I want enough elements so that my search is not trivial.

r = range(0,10**7,5)
mylist = list(r)
mytuple = tuple(r)
myset = set(r)

type(myset)

set

0 in mylist

True

1 in mylist

False

The following is an example of what’s called a Jupyter “magic”. This is not native to Python, it’s something that works specifically because we’re in Jupyter notebook.

%%timeit
0 in mylist

34.1 ns ± 0.712 ns per loop (mean ± std. dev. of 7 runs, 10,000,000 loops each)

%%timeit
0 in mytuple

31.9 ns ± 0.602 ns per loop (mean ± std. dev. of 7 runs, 10,000,000 loops each)

%%timeit
0 in myset

37.4 ns ± 0.464 ns per loop (mean ± std. dev. of 7 runs, 10,000,000 loops each)

You might be surprised because I told you that searching a set is much faster than searching a tuple or a list. But, what’s happening here is more a function of the fact that 0 appears in all of my variables, and it’s the first entry of mylist and mytuple. We’ll get a better sense of how fast things are by searching for something that never appears.

%%timeit
1 in mylist

22.7 ms ± 361 µs per loop (mean ± std. dev. of 7 runs, 10 loops each)

%%timeit 
1 in mytuple

20.4 ms ± 323 µs per loop (mean ± std. dev. of 7 runs, 10 loops each)

%%timeit
1 in myset

37.8 ns ± 0.521 ns per loop (mean ± std. dev. of 7 runs, 10,000,000 loops each)

Why so fast? Part of the reason is that lists aren’t allowed as elements of sets. Searching a dictionary for keys is equally fast.

Constant List#

Mini Goal: Write a function that takes as input a natural number \(n\) and as output returns a length \(n\) list \([3,3, \dots, 3]\).

We’ve already seen programming fundamentals in ML. Now we just need to learn the syntax for python.

def f(n):
    mylist = []
    for i in range(n):
        mylist.append(3)

f(3)

Notice that we didn’t actually return anything in our function.

def f(n):
    mylist = []
    for i in range(n):
        mylist.append(3)
        return mylist

f(5)

[3]

f(4)

[3]

This is happening because return is inside of the for-loop.

def f(n):
    mylist = []
    for i in range(n):
        mylist.append(3)
    return mylist
